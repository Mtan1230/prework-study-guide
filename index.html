HTML

Hypertext Markup Language (HTML) defines the basic framework for the page and the elements we want to appear on it.

An element is a part of a webpage that holds content or performs a specific task.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Void element, or self-closing tags: <img>, <br>
  • Common semantic elements: <article>, <aside>, <details>, <figcaption>, <figure>, <footer>, <header>, <main>, <nav>, <section>

Attributes are used to provide additional information about an element. All elements -- including void elements -- can accept attributes.
  • Global Attributes: data-*, dir, lang, style, class, id
  • Non-Global Attributes: alt, href, placeholder, src, value

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

Common Git commands

  • Create directory: mkdir
  • Create files (and modify their metadata): touch
  • Copy files: cp
  • Move files: mv
  • Delete files or directories: rm/rmdir
  • Download files found at specific URLs: curl
  • git status: checks what branch we are currently on
  • git chechout -b branch-name: creates a new branch and switches to it
  • git add -A: adds the changed files in our working directory to the staging area of Git
  • git commit -m "": commits the changes in the staging area to the repository, with a "-m" message to describe the changes implemented
  • git pull origin main: pulls the base branch into the feature branch to make sure the current branch is up to date
